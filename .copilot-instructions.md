# Copilot / GitHub Account-specific Instructions

Purpose: Document Copilot behavior, account-model mappings, and safe defaults for AI edits when using GitHub Copilot or
Copilot Chat.

1. Account → Model mapping

- `github_business` (company): prefer `Claude Opus 4.6` (see `.ai-models.yaml`).
- `github_personal` (GitHub Pro): prefer `GPT-5 mini`.

2. Token-awareness and chunking

- Detect model from the active GitHub account. If active model has limited context, automatically chunk large files
 using `.ai-models.yaml` recommended_chunk_tokens and `overlap_tokens`.
- If a task will exceed model limits, ask the human or use fallbacks.

3. Fallbacks and degraded behaviour

- If `Claude Opus 4.6` is not available, fall back to `Claude Haiku 4.5` (then `GPT-5 mini`) and shorten chunk sizes.
- On fallback, include a short note in the change summary: "[AI-FALLBACK: Claude Opus 4.6 -> (fallback)]".

4. Safety and scope

- Use the repository `.AI-GUIDELINES.md` and `.instructions.md` as the ultimate policy source.
- Verify `pre-commit` locally when making edits. Do not bypass pre-commit.

5. Example: detecting account and choosing model (pseudocode)

```sh
if github_account == 'business':
  # business account prefers the enterprise Claude Opus 4.6 model
  model = 'claude_opus_4_6'
else:
  model = 'gpt_5_mini'
# then apply chunking per .ai-models.yaml
```

6. Commits & messages

- Add `[AI-ASSISTED]` to commit messages when changes were generated or modified by AI.
- If a fallback occurred, include `[AI-FALLBACK]` with brief reason.

7. Human gate

- Always present generated diffs for human review before committing.
