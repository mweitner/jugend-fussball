# Canonical prompt template for repository AI agents

System instructions (high-level, read-only):

- You are an assistant editing documentation in this repository.
- Always follow `.AI-GUIDELINES.md` and `.instructions.md`.
- Preserve sensitive data; do not invent credentials.

User prompt template (fill fields):

- task: short description of the requested change
- files: list of file paths to edit or analyze
- repo_context: short summary of repository intent (one sentence)
  Note: this is a **German-language repository** — all content uses German umlauts
  (`ä`, `ö`, `ü`, `ß`); preserve them in all edits and never flag them as encoding errors.
- model: (optional) requested model name or leave blank to use account default
- max_chunk_tokens: (optional) override recommended chunk size
- precommit: true/false — whether to run pre-commit after edits

Example prompt body:

TASK: "Fix MD040 and MD013 violations in listed files"
FILES:

- doc-engine/source/foo.md
- doc-engine/source/bar.md
REPO_CONTEXT: "LPO system architecture docs; prefer 119-char line width; keep tables padded"
MODEL: "gpt_5_mini"
MAX_CHUNK_TOKENS: 8000
PRECOMMIT: true

Behavior rules:

- If `MODEL` is provided, validate it against `.ai-models.yaml`. If unsupported, return a clear error and suggest the
 account default. If `Claude Opus 4.6` is requested, ensure the human has approved enterprise-cost usage before
 proceeding.
- For tasks that touch > recommended_chunk_tokens, split work into chunks, process each chunk, and include a short
 merge plan for the human reviewer.
- Do not change `.pre-commit-config.yaml` or `.markdownlint.yaml` without explicit human approval.
- Include `[AI-ASSISTED]` in suggested commit messages.

Output format (use exactly):

- summary: one-line summary of edits
- files_changed: list of files modified
- diffs: git diff style blocks for each file
- commands: suggested shell commands to run for verification (e.g., `pre-commit run --all-files`)
- notes: optional extra guidance for reviewer

Chunking guidance:

- Use `.ai-models.yaml` values to decide chunk size and overlap.
- Prefer semantic splits at paragraph or section boundaries.

End of template
